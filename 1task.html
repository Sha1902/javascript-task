Semantic Elements

Use of semantic elements to describe the role of content

Examples of Semantic elements

  • header
  • footer
  • article

Examples of non semantic elements

  • div
  • span

Example usage:

  • <article> - independent content (blog post, news item)
  • <aside> - defines content should be indirectly related to the surrounding content
  • <section> - defines a section in a document
  • <header> & <footer> - page or section headers/footers

Text and Inline Elements

Block-level vs inline - block elements start on a new line (<p>,<h1>...); inline elements dont start on a new line , they stay in the flow (<span>,<a>).

Paragraphs and text semantics

<p> for paragrahs. Text-level semantics include:
  • <strong> for important text (usually bold)
  • <em> for emphasis (usally italic word)
  • <small> for fine print
  • <attr title='...'> for abbreviations
  • <mark> to highlight text
  • <code>,<pre> for code;<kbd> for keyboard input
